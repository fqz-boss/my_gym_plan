# 微信小程序部署指南

本仓库的小程序代码在 **`miniprogram/`** 目录。与 H5（`web/`，经 Cloudflare Pages 部署，见 [DEPLOY-D1.md](./DEPLOY-D1.md)）是并列关系：小程序不通过 `wrangler` 部署，需在微信公众平台完成体验版与正式发布。

---

## 一、前置条件

1. **微信账号**  
   可正常登录 [微信公众平台](https://mp.weixin.qq.com)（小程序类型）。

2. **小程序 AppID**  
   - 若已注册：在 **开发 → 开发管理 → 开发设置** 中查看 **AppID（小程序 ID）**。  
   - 若未注册：在 [微信公众平台](https://mp.weixin.qq.com) 注册 **小程序** 类型账号，按流程完成主体信息；注册成功后即可获得 AppID。

3. **开发工具**  
   安装 [微信开发者工具](https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html)（建议 Windows 使用稳定版，并保持更新）。

4. **权限说明**  
   上传代码、设置体验者、提交审核、发布，通常需要 **小程序管理员** 或已被授权的 **开发者** 身份。若你只有 **体验者**，只能扫体验版二维码，不能发版。

---

## 二、本地配置（首次必做）

### 1. 用微信开发者工具打开项目

1. 打开微信开发者工具。  
2. 选择 **导入** / **+** 新建项目。  
3. **目录** 选到本仓库的 **`miniprogram` 文件夹**（注意不是仓库根目录，与 `miniprogram/pages` 等同级的那一层）。  
4. **AppID**：填入你的小程序 AppID。  
   - 若仅为本地试跑、暂不关联线上小程序，可暂时选 **测试号**；正式发布前需改为真实 AppID 并在 `project.config.json` 中一致配置。

### 2. 填写 `project.config.json` 中的 AppID（推荐）

项目内 [`miniprogram/project.config.json`](../miniprogram/project.config.json) 中 `appid` 字段需与你的小程序一致，否则协作或 CI 同步时容易混淆：

```json
"appid": "你的AppID"
```

**注意**：AppID 属于应用标识，若入 Git 的仓库为公开仓库，可按团队规范选择是否用占位符 + 本地覆盖，避免将敏感配置提交到公网；私有仓库一般可直接提交。

### 3. 编译与预览

1. 在开发者工具中点击 **编译**，确认无报错。  
2. 点击 **预览**，用微信扫二维码，在真机上看页面与基础能力是否正常。

完成 `config.js` 中 `apiBase` 与合法域名后，在模拟器/真机走通「训练保存 → 记录列表 → 编辑」即认为联调就绪。

---

## 三、上传体验版（给体验者测试）

1. 在微信开发者工具中登录（与有该小程序 **开发者权限** 的微信一致）。  
2. 点击右上角 **上传**。  
3. 填写 **版本号**、**项目备注**（如「推拉腿 PPL 初版」），确认上传。  
4. 登录 [微信公众平台](https://mp.weixin.qq.com) → **管理 → 版本管理** → **开发版本**，可看到刚上传的版本。

### 选为体验版并加体验者

1. 在 **开发版本** 列表中，对某一版本选 **选为体验版**。  
2. **成员管理** 中确认体验者已添加为 **体验成员**；体验者用微信扫 **体验版** 的二维码即可安装（无需审核）。

体验版仅用于小范围测试，**不是** 面向所有用户线上环境。

---

## 四、提交审核与正式发布

1. 在 **版本管理** 中，将已通过测试的 **开发版本** 提审：  
   - 填写 **审核信息**、**服务类目**（与小程序实际功能一致，否则易驳回）。  
   - 按微信要求提供 **用户隐私保护指引**、**接口使用说明** 等（若已调用位置、剪贴板等需如实申报）。

2. 审核通过后，在 **版本管理** 中点击 **发布**，线上用户即可在微信内搜索或扫码使用（取决于你的推广方式）。

3. 版本号、更新说明会展示给用户，建议每次发版在备注里写清变更点，便于回滚与排查。

常见驳回原因：类目选错、隐私政策不完整、无实质功能、截图与说明不符。首次发版前建议阅读[小程序运营规范](https://developers.weixin.qq.com/miniprogram/product/)。

---

## 五、若小程序请求 H5/自建后端（域名白名单）

当前仓库内小程序示例如未使用 `wx.request` 等访问你的线上接口，可略读本节；**一旦在小程序里请求 `https` 接口**，必须在微信公众平台配置合法域名。

1. 登录 [微信公众平台](https://mp.weixin.qq.com) → **开发 → 开发管理 → 开发设置 → 服务器域名**。  
2. **request 合法域名**：填写你 H5/Worker 的域名，例如 `https://my-gym-plan.pages.dev` 或你绑定的**自定义域名**（须备案规则以微信文档为准，域名须 HTTPS）。  
3. 本仓库 H5 与 D1 部署见 [DEPLOY-D1.md](./DEPLOY-D1.md)；**小程序里请求的 URL 需与这里配置的域名一致**（含子域名）。  
4. 若使用 **web-view** 嵌 H5，还需在 **业务域名** 中配置相应域名。  
5. 开发者工具里可在 **详情 → 本地设置** 中勾选 **不校验合法域名** 仅作本地调试；**真机预览与正式环境一律校验**。

**说明**：`localhost` 不能作为线上合法域名；联调可借助内网穿透域名（同样需配进合法域名）或使用测试环境域名。

---

## 六、与本仓库其他部分的关系

| 部分 | 作用 |
|------|------|
| `miniprogram/` | 微信小程序源码，用微信开发者工具打开并上传。 |
| `web/` + Cloudflare Pages + D1 | H5 与数据接口部署，见 [DEPLOY-D1.md](./DEPLOY-D1.md)。 |
| `wrangler.toml` | 与 Pages/D1 绑定，**不** 用于发小程序。 |

在小程序中复用与 H5 相同的后端时，需在同一小程序中配置好 **服务器域名**，并在代码里使用经备案/允许的 HTTPS 地址（参考微信官方[网络](https://developers.weixin.qq.com/miniprogram/dev/framework/ability/network.html)说明）。

---

## 七、运维与回滚

- **回退版本**：在 **版本管理** 中可对旧线上版本做回退（有次数与条件限制，以平台当前说明为准）。  
- **分阶段发布**：支持按比例向部分用户放新版本，降低风险。  
- **数据与 API**：若训练数据走 Cloudflare，数据库与 H5 侧需同步维护；小程序端异常多为域名未配置、证书错误或接口跨域/鉴权问题（小程序侧通常不是 CORS，而是域名未在后台登记）。

---

## 八、清单（发版前核对）

- [ ] `miniprogram/project.config.json` 中 `appid` 正确。  
- [ ] 微信开发者工具 **上传** 成功，版本号与备注清晰。  
- [ ] 体验者已验证核心流程。  
- [ ] 已选类目、隐私与审核材料齐全。  
- [ ] 若有 `wx.request` 等，**服务器域名** 已包含所有线上 API 根域名。  
- [ ] 与 H5 共用的 API 在测试环境、生产环境均已自测通过。

按上述步骤即可完成从本地开发、体验版、审核到正式上线的完整部署流程。若你后续将 `miniprogram` 接入本仓库的 Pages API，建议在本文「五」中补充你实际使用的域名，并在 [DEPLOY-D1.md](./DEPLOY-D1.md) 中保持 H5/Worker 地址与小程序配置一致。
